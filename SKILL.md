# Component Audit Tool - Claude Skill

## 功能说明

这是一个设计系统分析工具。上传你的设计稿截图，我会：

1. **识别所有 UI 组件** - 按钮、卡片、表单、导航等
2. **检查一致性** - 颜色、字体、间距、圆角的一致性评分
3. **找出重复** - 哪些组件可以合并或标准化
4. **评估系统成熟度** - 给出 1-10 的评分
5. **提供优化建议** - 优先级、工作量、预期收益
6. **提取设计 Token** - 色彩系统、字体系统

## 使用方式

### 步骤 1：上传设计稿
- 直接拖拽或粘贴设计稿截图到对话框
- 支持 PNG、JPG、WebP 格式
- 推荐宽度 1200px+ 的高清设计稿

### 步骤 2：请求分析
在 Claude 对话框输入：
```
请帮我分析这个设计稿中的组件一致性。使用 Component Audit Tool
```

### 步骤 3：获取报告
你会得到结构化的分析结果，包括：
- 📊 组件数量和一致性评分
- 🎨 所有识别的组件清单
- ⚠️ 发现的问题（严重程度分级）
- 💡 优化建议（优先级排序）
- 🎨 色彩系统提取
- 🔤 字体系统分析

## 分析指南

### 我会特别注意：
- **一致性** - 同类组件的尺寸、颜色、间距是否统一
- **可复用性** - 有哪些相似的组件可以合并
- **设计 Token** - 提取可重复使用的设计规范
- **命名规范** - 组件命名是否遵循体系
- **优先级** - 高优先级的问题和建议放在前面

### 我会给出：
1. **具体的十六进制颜色值** - 而不是模糊的"蓝色"
2. **明确的像素数值** - 如果能看出间距和尺寸
3. **位置引用** - 例如"Header 中的 3 个按钮"
4. **对标准组件库** - 例如 Material Design、Ant Design 的对比

## 使用场景

✅ **评审设计稿** - 在设计评审前检查一致性  
✅ **优化组件库** - 找出需要统一的地方  
✅ **交接文档** - 为开发生成清晰的设计规范  
✅ **学习分析** - 理解好的设计系统长什么样
